# init-case — Инициализация дела

Когда пользователь запускает эту команду, выполни следующее:

## 1. Собери информацию о деле

Спроси Сюзерена (пользователя-юриста) о деле в свободной форме. Тебе нужно узнать:

- **Номер дела** (например, А33-12345/2026). Если дело ещё не возбуждено — оставь пустым.
- **Суд** (название, например «Арбитражный суд Красноярского края»).
- **Стороны**: кто истец, кто ответчик (название организации, ИНН/ОГРН если есть).
- **Кто наш клиент** (какая из сторон).
- **Суть спора** (кратко: «взыскание задолженности по договору поставки»).
- **Судья** (если известен).
- **Дата подачи иска**, **дата следующего заседания** (если известны).
- **Данные представителя** (ФИО, доверенность, контакты).

Не требуй всё сразу — заполни что дано, остальное можно дополнить позже. Если пользователь дал текстовое описание — извлеки всё, что можешь.

## 2. Создай структуру папок

В текущей рабочей директории создай:

```
.vassal/
├── case.yaml
├── index.yaml
├── raw/
├── mirrors/
├── analysis/
├── positions/
├── hearings/
└── history.md

Входящие документы/        ← сюда пользователь кладёт файлы для обработки
На удаление/               ← корзина (rm не работает в Cowork)
Материалы от клиента/
```

Используй bash для создания директорий:
```bash
mkdir -p .vassal/{raw,mirrors,analysis,positions,hearings}
mkdir -p "Материалы от клиента"
mkdir -p "Входящие документы"
mkdir -p "На удаление"
```

## 3. Заполни case.yaml

Прочитай формат из `shared/case-schema.yaml` (рядом с этой командой в плагине). Заполни все известные поля, неизвестные оставь как `null`.

Важно:
- Каждой стороне присвой стабильный `party_id` в формате `{role}-01` (например, `plaintiff-01`, `defendant-01`).
- Поле `our_client.party_id` ссылается на `party_id` нашего клиента.
- Оценочные поля (досудебный порядок, подсудность) пока не заполняй — это задача `legal-review`.
- `version: 2` — обязательно.

## 4. Создай пустой index.yaml

```yaml
version: 2
last_updated: {текущая дата и время ISO}

documents: []

bundles: []

next_id: 1
next_bundle_id: 1
```

## 5. Создай history.md

```markdown
# История действий

## {ГГГГ-ММ-ДД}

### {ЧЧ:ММ} — Инициализация дела
- Создана карточка дела {номер дела или "без номера"}
- Стороны: {истец} (истец) vs {ответчик} (ответчик)
- Наш клиент: {название} ({роль})
- Представитель: {ФИО представителя}
```

## 6. Покажи результат

Выведи Сюзерену краткую сводку:
- Номер дела и суд
- Стороны и наш клиент
- Какие поля остались пустыми (чтобы знать, что дополнить)
- Подскажи следующий шаг: «Положи материалы клиента в папку **Входящие документы/** — я обработаю их через intake.»

## Проверки

- Если `.vassal/case.yaml` уже существует — **не перезаписывай**. Предупреди: «Дело уже инициализировано. Хочешь обновить карточку или создать заново?»
- Если пользователь не дал достаточно информации даже для минимальной карточки (нет ни сторон, ни сути спора) — объясни, что нужно для старта, и попроси описать дело.
