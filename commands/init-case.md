# init-case — Инициализация дела

## Контекст запуска

Команда запускается в папке, куда пользователь уже сложил сырые материалы дела. Текущая рабочая директория и есть корень дела: не нужно создавать дополнительную подпапку, спрашивать её имя или менять `cwd`.

Скилл сам:

1. Разворачивает в `cwd` скелет структуры (`.vassal/`, `Входящие документы/`, `Материалы от клиента/`).
2. Сгребает сырьё из корня в `Входящие документы/`.
3. Запускает `intake`.
4. Извлекает кандидатов на поля карточки дела из зеркал и индекса.
5. Спрашивает только недостающие поля.

## Последовательность фаз

0. **Preflight** — проверка, что дело ещё не инициализировано.
1. **Scaffold** — создание папок, перенос входящих файлов, заглушки `case.yaml`, `index.yaml`, `history.md`.
2. **Intake** — обработка файлов из `Входящие документы/`.
3. **Extract candidates** — извлечение кандидатов на поля карточки из зеркал.
4. **Questionnaire** — опрос только по недостающим полям.
5. **Finalize** — запись `case.yaml`, финальная запись в `history.md`, краткая сводка.

## Жёсткие правила

- Папка дела — это текущая директория.
- До завершения `intake` и чтения зеркал нельзя задавать вопросы о полях дела.
- Вопросы пользователю задаются только через `AskUserQuestion`.
- Все тексты вопросов — только по-русски.

## Фаза 0 — Preflight

1. Проверь, существует ли `.vassal/case.yaml`.
2. Если существует — сообщи, что дело уже инициализировано, и остановись.
3. Определи `[PLUGIN_ROOT]` как корень установленного плагина.
4. Если зависимости ещё не подготовлены в текущей сессии — выполни `bash [PLUGIN_ROOT]/scripts/setup.sh`.

## Фаза 1 — Scaffold

Создай структуру:

```bash
mkdir -p .vassal/{raw,mirrors,analysis,positions,hearings,plans,work}
mkdir -p "Материалы от клиента"
mkdir -p "Входящие документы"
```

Затем перенеси все несистемные файлы из корня во `Входящие документы/`.

## Фаза 2 — Intake

1. Проверь содержимое `Входящие документы/`.
2. Если папка пуста — спроси, ждать ли дозагрузки файлов или продолжить без документов.
3. Если файлы есть — прочитай `skills/intake/SKILL.md` и выполни его контракт целиком:
   - preview;
   - подтверждение пользователя;
   - apply;
   - verify.
4. После успешного `intake` должны появиться записи в `.vassal/index.yaml` и зеркала в `.vassal/mirrors/`.

## Фаза 3 — Extract candidates

1. Прочитай `.vassal/index.yaml`.
2. Прочитай frontmatter и начало ключевых зеркал из `.vassal/mirrors/`.
3. Собери кандидатов на поля карточки дела: стороны, номер дела, суд, судью, категорию, предмет, даты, суммы, представителя, досудебный порядок.

## Фаза 4 — Questionnaire

1. Сначала спроси, кто наш клиент.
2. Затем пакетами до четырёх вопросов добери только поля без уверенного кандидата.
3. Не спрашивай то, что уже надёжно извлечено из материалов.

## Фаза 5 — Finalize

1. Запиши итоговый `case.yaml`.
2. Добавь запись в `.vassal/history.md`.
3. Покажи краткую сводку: какие документы приняты, какие поля определены автоматически, что осталось на ручную проверку.
