# analyze-hearing -- Анализ транскрипции заседания

Когда пользователь запускает эту команду с "$ARGUMENTS":

1. Прочитай скилл `skills/analyze-hearing/SKILL.md` и следуй его инструкциям.
2. Если `$ARGUMENTS` содержит текст транскрипции -- используй его напрямую.
3. Если `$ARGUMENTS` содержит путь к файлу -- прочитай файл.
4. Если аргументов нет -- проверь «Входящие документы/» на наличие файлов транскрипции, или спроси Сюзерена.
5. Убедись, что `.vassal/case.yaml` существует.
6. Выполни `skills/analyze-hearing/SKILL.md` по схеме: сбор и нормализация -> preview -> confirm -> apply -> verify.
7. Apply-фазу выполняй только через `Task(model=opus)` по контракту 3.1c; не используй Sonnet-subagent и не переключайся на `.docx`.
8. Перед сохранением `analysis.md` проверь валидации из `skills/analyze-hearing/SKILL.md`, включая обязательный якорь `## Ход заседания` и минимум ещё одну предметную секцию.
