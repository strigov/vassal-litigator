# Catalog — Каталогизация документов дела

Генерирует человекочитаемую таблицу всех документов дела в формате `.xlsx` и обогащает `index.yaml` описаниями (summary) там, где их не хватает.

## Предварительные условия

- Существует `.vassal/index.yaml` с записями о документах
- Существуют md-зеркала в `.vassal/mirrors/`
- Если индекс пуст — сообщи Сюзерену, что сначала нужно загрузить документы через intake

## Алгоритм

### Шаг 1 — Загрузка данных

1. Прочитай `.vassal/case.yaml` (контекст дела).
2. Прочитай `.vassal/index.yaml` (все документы).

### Шаг 2 — Обогащение описаний

3. Для каждого документа, где `summary` пуст или слишком короткий (< 20 символов):
   - Прочитай md-зеркало (`.vassal/mirrors/{id}.md`)
   - Делегируй Haiku-субагенту (model: "haiku") задачу: «Прочитай текст документа и составь краткое описание содержания в 1-3 предложениях. Укажи ключевые суммы, даты, стороны, обязательства.»
   - Обнови `summary` в index.yaml

4. Для каждого документа также убедись, что заполнены: `seal`, `signature`, `completeness`, `quality`. Если нет — извлеки из метаданных зеркала или прочитай начало зеркала.

### Шаг 3 — Генерация таблицы

5. Сгенерируй `Таблица документов.xlsx` в корне папки дела. Используй скилл xlsx или python (openpyxl).

Колонки таблицы:

| Колонка | Источник | Пример |
|---------|----------|--------|
| № | порядковый номер | 1 |
| Дата | `date` из index.yaml | 01.06.2025 |
| Тип | `type` | Договор |
| Отправитель → Адресат | `parties` | ООО «Ромашка» → ООО «Лютик» |
| Название | `title` | Договор поставки №47 |
| Краткое содержание | `summary` | Поставка стройматериалов, 1.5 млн руб. |
| Печать / Подпись | `seal`, `signature` | Печать + подпись |
| Полнота | `completeness` | Полный |
| Качество | `quality` | Читаемый |
| Файл | `file` (относительный путь) | Материалы от клиента/Договоры/... |

Форматирование таблицы:
- Заголовок выделен жирным, закреплён (freeze panes)
- Автоширина колонок
- Группировка по типам документов (договорные, переписка, процессуальные, платёжные, прочие) — разделяй пустой строкой с заголовком группы
- Если есть бандлы — документы одного бандла идут подряд, приложения с отступом

### Шаг 4 — Обновление и отчёт

6. Сохрани обновлённый `index.yaml`.
7. Добавь запись в `.vassal/history.md`.
8. Покажи Сюзерену:
   - Сколько документов в реестре
   - Сколько описаний добавлено/обновлено
   - Есть ли документы с `needs_manual_review: true`
   - Ссылку на файл таблицы

## Идемпотентность

Повторный запуск catalog не создаёт дублей — он перезаписывает `Таблица документов.xlsx` и обновляет только пустые/неполные `summary` в index.yaml. Существующие описания не перезаписываются.

## Граничные случаи

- Документ без md-зеркала → пропусти при генерации summary, включи в таблицу с пометкой «Зеркало отсутствует»
- Очень много документов (>100) → генерируй summary пакетами по 10, чтобы не перегрузить контекст
- Таблица уже существует → перезаписать (предупредив Сюзерена)
