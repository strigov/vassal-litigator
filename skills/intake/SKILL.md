# Intake — Приём материалов клиента

Этот скилл обрабатывает пакет файлов от клиента: сохраняет оригиналы, извлекает текст, переименовывает, раскладывает по тематическим папкам, создаёт md-зеркала и обновляет индекс дела. Работает по принципу **preview → apply**: сначала показывает план, выполняет только после подтверждения Сюзерена (юриста).

## Предварительные условия

- Дело инициализировано: существует `.vassal/case.yaml`. Если нет — предложи запустить `/vassal-litigator:init-case`.
- Существует папка `Входящие документы/`. Если нет — создай.
- Установлены зависимости для OCR. Запусти `scripts/setup.sh` если ещё не запускал в этой сессии.

## Алгоритм

### Фаза 1 — Сохранение оригиналов

1. Прочитай `.vassal/case.yaml` — тебе нужны стороны и предмет спора для контекста.
2. **Источник файлов — `Входящие документы/`**. Это единственное место, откуда intake забирает файлы. Если папка пуста — сообщи: «Папка "Входящие документы/" пуста. Положите туда файлы и запустите intake снова.»
3. Скопируй все файлы из `Входящие документы/` «как есть» в `.vassal/raw/intake-{ГГГГ-ММ-ДД}/`. Используй `cp`, не `mv` — на этом этапе оригиналы во «Входящие документы/» ещё не трогаем (удаление — фаза 5, после подтверждения).
4. Зафиксируй для каждого файла: исходное имя (`origin_name`), дату получения, пакет приёма.

### Фаза 2 — Извлечение и анализ

5. **Рекурсивный обзор**: получи список всех файлов (включая вложенные папки).
6. **Распаковка архивов**: zip, rar, 7z, tar.gz — распакуй в подпапку с именем архива. Добавь содержимое в список обработки.
7. **OCR pipeline** для каждого файла:
   - PDF с текстовым слоем → извлеки текст программно (`scripts/extract_text.py` или Read tool)
   - DOCX → парси через Read tool или python-docx
   - Изображения / сканы → `scripts/extract_text.py` (tesseract), при низком качестве — Read tool для визуального анализа (LLM fallback)
   - Для каждого определи `extraction_method` и `confidence`
8. **Анализ имён файлов** — используй подход file-renamer (три категории):
   - Полная информация из имени → сразу можно переименовать
   - Частичная → дочитать из содержимого начало документа (первая страница / 2000 символов)
   - Имя бесполезно → полный анализ начала через чтение
9. **Категоризация** — определи для каждого файла:
   - Тип документа (из таксономии: договор, претензия, платёжное-поручение и т.д. — см. `shared/conventions.md`)
   - Дату документа
   - Стороны (отправитель, адресат)
   - В какую подпапку `Материалы от клиента/` поместить

Делегируй Haiku-субагентам (model: "haiku") массовую обработку: извлечение текста, первичную категоризацию, переименование. Sonnet оставь себе для принятия решений о систематизации.

### Фаза 3 — Preview

10. Покажи Сюзерену план:

```
Пакет: intake-2026-03-26 (15 файлов)

Материалы от клиента/Договоры/
  скан_001.pdf -> 2025-06-01 ООО Ромашка Договор поставки №47.pdf
  доп_согл.docx -> 2025-08-15 ООО Ромашка Дополнительное соглашение №1.docx

Материалы от клиента/Переписка/
  претензия.pdf -> 2025-09-15 ООО Ромашка Претензия.pdf
  ответ.pdf -> 2025-10-01 ООО Лютик Ответ на претензию.pdf

Материалы от клиента/Платёжные документы/
  п_п_123.pdf -> 2025-06-05 ООО Лютик Платёжное поручение №123.pdf

ВНИМАНИЕ — требуют проверки:
  111.txt -> 0000-00-00 Неизвестный Документ.txt (не удалось определить)
  скан_плохой.jpg -> 2025-07-01 ООО Ромашка Документ.jpg (низкое качество OCR)

После обработки из «Входящие документы/» будут перемещены в «На удаление/»:
  скан_001.pdf, доп_согл.docx, претензия.pdf, ответ.pdf, п_п_123.pdf, ...
  (оригиналы сохранены в .vassal/raw/intake-2026-03-26/)
```

Типовые подпапки: `Договоры/`, `Переписка/`, `Платёжные документы/`, `Учредительные/`, `Иное/`. Если документы не укладываются — предложи кастомные.

11. Дождись подтверждения. Если Сюзерен корректирует — внеси правки.

### Фаза 4 — Apply

12. Создай подпапки в `Материалы от клиента/`.
13. Скопируй файлы из исходного расположения в целевые папки с новыми именами.
14. Для каждого файла создай md-зеркало в `.vassal/mirrors/`:
    - Формат: см. `shared/mirror-template.md`
    - ID: `doc-{NNN}` (бери `next_id` из `index.yaml`, инкрементируй)
    - Делегируй создание Haiku-субагентам (model: "haiku")
15. Обнови `.vassal/index.yaml`:
    - Добавь запись для каждого файла со всеми полями (см. `shared/index-schema.yaml`)
    - Заполни `origin` (исходное имя, дата получения, пакет)
    - Установи `mirror_stale: false`, `needs_manual_review` по качеству OCR
    - Обнови `next_id`
16. Добавь запись в `.vassal/history.md`.

### Фаза 5 — Очистка «Входящие документы/»

17. Перемести из `Входящие документы/` все файлы, которые были успешно обработаны в фазе 4, в папку `На удаление/`.
    - **Не используй `rm`** — файловая система Cowork не поддерживает удаление. Вместо этого:
      ```bash
      cp "Входящие документы/файл.pdf" "На удаление/файл.pdf"
      : > "Входящие документы/файл.pdf"
      ```
    - Побайтовые оригиналы уже лежат в `.vassal/raw/intake-{дата}/`.
    - Если в `Входящие документы/` были вложенные папки — обрабатывай рекурсивно, но только обработанные файлы.
    - Если какой-то файл не был обработан (ошибка, пропущен) — **оставь его** во «Входящие документы/».
    - При совпадении имён в `На удаление/` — добавь суффикс `(2)`, `(3)`.
    - После очистки сообщи: «Обработанные файлы перемещены в На удаление/. Оригиналы → .vassal/raw/intake-{дата}/».

## Формат имени файла

```
ГГГГ-ММ-ДД Отправитель Описание.расширение
```

Примеры:
- `2025-06-01 ООО Ромашка Договор поставки №47.pdf`
- `2025-09-15 ООО Ромашка Претензия.pdf`
- `2025-06-05 ООО Лютик Платёжное поручение №123.pdf`

Убирай кавычки из названий: `ООО «Ромашка»` → `ООО Ромашка`. Расширение файла никогда не меняй.

## Идемпотентность

Если intake уже запускался (есть файлы в `Материалы от клиента/` и записи в `index.yaml`):
- Проверь, какие файлы из нового пакета уже обработаны (по `origin.name` + `origin.intake_batch`)
- Обработай только новые
- Сообщи: «N файлов уже были обработаны ранее, пропускаю»

## Граничные случаи

- Пустой архив → пропусти, сообщи
- Файл нулевого размера → пропусти, сообщи
- Файл с повреждённой кодировкой → пометь `needs_manual_review: true`, создай зеркало с тем, что удалось извлечь
- Очень большой файл (>50 страниц) → обработай первые 10 страниц для зеркала, в зеркале пометь `[... документ содержит N страниц, извлечены первые 10 ...]`
- Дубликат имени после переименования → добавь суффикс `(2)`, `(3)`
