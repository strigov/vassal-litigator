# Intake — Приём материалов клиента

Скилл работает по принципу preview→apply: preview строит Claude-main (read-only, никаких файловых операций), apply выполняет Codex medium (--write).

## Предварительные условия

- Дело инициализировано: существует `.vassal/case.yaml`. Если нет — предложи запустить `/vassal-litigator:init-case`.
- Существует папка `Входящие документы/`. Если нет — **не создавай** в preview-фазе; Codex создаст её в apply-фазе. Сообщи Сюзерену: «Папка "Входящие документы/" не найдена — она будет создана автоматически при подтверждении.»
- Установлены зависимости для OCR. Запусти `[PLUGIN_ROOT]/scripts/setup.sh` если ещё не запускал в этой сессии.

## Фаза 1 — Preview (Claude-main, read-only)

Claude-main ДО подтверждения Сюзерена:

1. Прочитай `.vassal/case.yaml` — стороны и предмет.
2. Получи список файлов в `Входящие документы/` только по именам и размерам. Используй только Read/Glob. Не делай OCR, не запускай скрипты и не трогай содержимое файлов.
3. На основе имён файлов и размеров предложи план: переименование по шаблону `ГГГГ-ММ-ДД Отправитель Описание.расш`, целевые папки в `Материалы от клиента/`, назначенные `doc-ID`.
4. Прочитай поле `next_id` из `.vassal/index.yaml` и назначь IDs в плане, начиная с `next_id`. Инкрементируй только внутри плана; в `.vassal/index.yaml` ничего не пиши.
5. Покажи Сюзерену preview:
   - Таблица: исходное имя → новое имя → целевая папка → `doc-ID`
   - Список файлов `ВНИМАНИЕ — требуют проверки` (непонятное имя, нулевой размер, неизвестное расширение)
   - Что будет перемещено в `На удаление/` после обработки

ЗАПРЕЩЕНО в этой фазе: любые `cp`, `mv`, `mkdir`, python-скрипты, запись в `.vassal/`, изменение любых файлов.

## Фаза 2 — Подтверждение

Дождись ответа Сюзерена. Если он корректирует план — обнови preview и покажи снова. Если отменяет операцию — остановись, ничего не делай.

## Фаза 3 — Apply (Codex medium --write)

После подтверждения:

1. Прочитай `skills/codex-invocation/SKILL.md` (раздел `Как Claude-main собирает prompt`).
2. Собери prompt: прочитай `prompts/_preamble.md` → прочитай `prompts/file-executor-intake.md` → подставь переменные:
   - `case_root`: абсолютный путь к делу
   - `plugin_root`: абсолютный путь к плагину по 3-tier fallback из `skills/codex-invocation/SKILL.md`
   - `batch_name`: `intake-{ГГГГ-ММ-ДД}`
   - `plan_body`: весь согласованный план текстом, включая таблицу файлов и назначенные IDs
   - `extra_constraints`: пусто, если нет дополнительных ограничений
   - `report_contract`: стандартный блок отчёта, скопированный из раздела `Формат отчёта` в `prompts/_preamble.md`
3. Проверь, что все плейсхолдеры раскрыты:
   `grep -c "{{" prompt.txt`
   Результат должен быть `0`.
4. Диспатч задачу через `codex-companion.mjs`:
   `task --background --write --effort medium "..."`
5. Мониторь статус по шаблону `until/case loop` из `skills/codex-invocation/SKILL.md` с `sleep 25`.
6. Получи итог через fetch/result.

## Фаза 4 — Верификация (Claude-main)

1. Прочитай отчёт Codex.
2. Проверь, что `FILES_CREATED` совпадает с планом, `.vassal/index.yaml` читается без ошибок, а в отчёте нет упоминаний операций вне согласованного плана.
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-intake.md`:
   промпт + отчёт Codex.
4. Покажи Сюзерену резюме: сколько файлов обработано, сколько записей добавлено в индекс, есть ли `needs_manual_review`.

## Fallback без Codex

Если Codex недоступен (`companion.mjs` отсутствует, `ECONNREFUSED`, ошибка запуска):

- Предупреди Сюзерена: `Codex недоступен, выполняю apply-фазу напрямую (медленнее, дороже по токенам)`.
- Выполни apply-фазу самостоятельно по шагам из `prompts/file-executor-intake.md`.
- OCR и извлечение текста запускай через:
  `python3 [PLUGIN_ROOT]/scripts/extract_text.py`
- Лог сохрани в `.vassal/codex-logs/{дата-время}-intake-fallback.md`.

## Формат имени файла

`ГГГГ-ММ-ДД Отправитель Описание.расширение`

Примеры:

- `2025-06-01 ООО Ромашка Договор поставки №47.pdf`
- `2025-09-15 ООО Ромашка Претензия.pdf`
- `2025-06-05 ООО Лютик Платёжное поручение №123.pdf`

Убирай кавычки из названий: `ООО «Ромашка»` → `ООО Ромашка`. Расширение файла никогда не меняй.

## Идемпотентность

Если intake уже запускался (есть файлы в `Материалы от клиента/` и записи в `index.yaml`):

- Проверь, какие файлы из нового пакета уже обработаны по `origin.name` + `origin.intake_batch`
- Обработай только новые
- Сообщи: `N файлов уже были обработаны ранее, пропускаю`
