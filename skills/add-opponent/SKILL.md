# add-opponent -- Прием документов оппонента

preview→apply: preview Claude-main (read-only), apply Codex medium (--write).

## Предварительные условия

- Дело инициализировано: `.vassal/case.yaml` существует.
- В `case.yaml` определены стороны (чтобы знать, кто оппонент).
- Существует папка `Входящие документы/`. Если нет -- **не создавай** её в preview-фазе; Codex создаст автоматически в apply-фазе. Сообщи Сюзерену: «Папка "Входящие документы/" не найдена -- она будет создана автоматически при подтверждении.»

## Фаза 1 -- Preview (Claude-main, read-only)

1. Прочитай `.vassal/case.yaml` и определи оппонента.
2. Прочитай `.vassal/index.yaml` для контекста дела.
3. Получи список файлов в `Входящие документы/` только по именам и размерам. Не делай OCR и не читай содержимое файлов.
4. По именам и контексту дела предложи:
   - тип основного документа оппонента: отзыв, ходатайство, пояснения, возражение и т.д.
   - название процессуальной папки по шаблону `{ГГГГ-ММ-ДД} {Сторона оппонента} - {тип документа}/`
   - план бандла: основной документ + приложения
   - новые имена файлов
   - `doc-ID` из `next_id`
5. Покажи Сюзерену preview:
   - список файлов по именам и размерам
   - предложенную процессуальную папку
   - план бандла
   - назначенные `doc-ID`
   - список `ВНИМАНИЕ -- требуют проверки`
6. Отдельно укажи, что после подтверждения:
   - будет создан пакет `opponent-{ГГГГ-ММ-ДД}`
   - оригиналы будут скопированы в `.vassal/raw/opponent-{ГГГГ-ММ-ДД}/`
   - файлы из `Входящие документы/` будут очищены через `cp` + `: >`

ЗАПРЕЩЕНО: `cp`, `mv`, OCR, python-скрипты, запись в `.vassal/`, изменение любых файлов.

## Фаза 2 -- Подтверждение

Дождись ответа Сюзерена. Если план изменён -- пересобери preview и покажи снова. Если операция отменена -- остановись без изменений.

## Фаза 3 -- Apply (Codex medium --write)

1. Прочитай `skills/codex-invocation/SKILL.md`.
2. Собери prompt из `prompts/file-executor-add-opponent.md`, подставь:
   - `case_root`
   - `plugin_root` по 3-tier fallback из `skills/codex-invocation/SKILL.md`
   - `batch_name`: `opponent-{ГГГГ-ММ-ДД}`
   - `plan_body`: согласованный план целиком, включая `bundle_id`, если он утверждён
   - `extra_constraints`: пусто
   - `report_contract`: из `prompts/_preamble.md`
3. Проверь:
   `grep -c "{{" prompt.txt`
   Результат должен быть `0`.
4. Диспатч:
   `codex-companion.mjs task --background --write --effort medium`
5. Мониторь выполнение по шаблону из `skills/codex-invocation/SKILL.md`.
6. Получи итог через fetch/result.

## Фаза 4 -- Верификация (Claude-main)

1. Прочитай отчёт Codex.
2. Проверь, что:
   - raw-копии созданы в `.vassal/raw/opponent-{дата}/`
   - зеркала созданы
   - `.vassal/index.yaml` валиден
   - бандл оформлен согласно плану
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-add-opponent.md`:
   prompt + отчёт Codex.
4. Покажи Сюзерену резюме: сколько файлов обработано, сколько записей добавлено, есть ли `needs_manual_review`.

## Фаза 5 -- Экспресс-анализ (Opus)

ВАЖНО: эта фаза остаётся на Opus, это не файловая работа.

1. Если основной документ оппонента -- процессуальный (отзыв, возражение, пояснения), предложи Сюзерену:
   `Сделать экспресс-анализ аргументов оппонента?`
2. Если да -- через Opus-субагент:
   - извлеки ключевые доводы оппонента
   - сопоставь с нашей позицией, если в деле уже есть позиционные документы
   - кратко ответь: что нового, что опасно, что легко парируется
   - предложи ответные действия по делу
   - сохрани результат в `.vassal/analysis/opponent-{дата}.md`
   - покажи Сюзерену краткую сводку в чате

## Формат имени файла

```
ГГГГ-ММ-ДД Отправитель Описание.расширение
```

Отправитель для документов оппонента -- название организации оппонента (без кавычек).

## Формат процессуальной папки

```
{ГГГГ-ММ-ДД} {Сторона} - {действие}
```

Сторона: «Истец», «Ответчик», «Третье лицо» (или конкретное название, если сторон несколько одного вида).

## Fallback без Codex

Если Codex недоступен:

- Предупреди Сюзерена: `Codex недоступен, выполняю apply-фазу напрямую`.
- Выполни только apply-фазу (Фаза 3) самостоятельно по шагам из `prompts/file-executor-add-opponent.md`.
- OCR и извлечение текста запускай через:
  `python3 [PLUGIN_ROOT]/scripts/extract_text.py`
- Лог сохрани в `.vassal/codex-logs/{дата-время}-add-opponent-fallback.md`.

Фаза 5 fallback-логикой не заменяется и продолжает выполняться отдельно, как Opus-анализ.

## Граничные случаи

- Документы от нескольких оппонентов в одном пакете -> создай отдельную папку для каждого.
- Документ от суда (определение, протокол) попал сюда по ошибке -> сообщи, предложи переквалифицировать.
- Оппонент прислал копии наших же документов -> пометь как дубликат, не создавай отдельную запись.
