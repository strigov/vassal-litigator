# add-evidence -- Прием дополнительных доказательств

preview→apply: preview Claude-main (read-only), apply Codex medium (--write).

## Предварительные условия

- Дело инициализировано: `.vassal/case.yaml` существует. Если нет -- предложи `/vassal-litigator:init-case`.
- Intake уже выполнен (есть записи в `index.yaml`). Если нет -- предложи сначала `/vassal-litigator:intake`.
- Существует папка `Входящие документы/`. Если нет -- **не создавай** её в preview-фазе; Codex создаст автоматически в apply-фазе. Сообщи Сюзерену: «Папка "Входящие документы/" не найдена -- она будет создана автоматически при подтверждении.»

## Фаза 1 -- Preview (Claude-main, read-only)

1. Прочитай `.vassal/case.yaml` и `.vassal/index.yaml`.
2. Получи список файлов в `Входящие документы/` только по именам и размерам. Не делай OCR, не запускай скрипты и не читай содержимое файлов.
3. На основе имён файлов, контекста дела и уже существующей структуры предложи план:
   - переименование по шаблону `ГГГГ-ММ-ДД Отправитель Описание.расш`
   - целевые папки:
     - `Материалы от клиента/{подпапка}/` для клиентских доказательств
     - `{ГГГГ-ММ-ДД} {Сторона} - {действие}/` для процессуальных файлов, если это следует из контекста
   - `doc-ID` из `next_id`
4. Покажи Сюзерену preview:
   - список файлов по именам и размерам
   - план переименований
   - целевые папки
   - назначенные `doc-ID`
   - список `ВНИМАНИЕ -- требуют проверки`
5. Отдельно укажи, что после подтверждения:
   - будет создан пакет `evidence-{ГГГГ-ММ-ДД}`
   - оригиналы будут скопированы в `.vassal/raw/evidence-{ГГГГ-ММ-ДД}/`
   - обработанные файлы из `Входящие документы/` будут перенесены в `На удаление/` через `cp` + `: >`

ЗАПРЕЩЕНО: `cp`, `mv`, OCR, python-скрипты, запись в `.vassal/`, изменение любых файлов.

## Фаза 2 -- Подтверждение

Дождись ответа Сюзерена. Если он корректирует план -- пересобери preview и покажи снова. Если отменяет операцию -- остановись без изменений.

## Фаза 3 -- Apply (Codex medium --write)

1. Прочитай `skills/codex-invocation/SKILL.md`.
2. Собери prompt из `prompts/file-executor-add-evidence.md`, подставь:
   - `case_root`
   - `plugin_root` по 3-tier fallback из `skills/codex-invocation/SKILL.md`
   - `batch_name`: `evidence-{ГГГГ-ММ-ДД}`
   - `plan_body`: согласованный план целиком
   - `extra_constraints`: пусто
   - `report_contract`: из `prompts/_preamble.md`
3. Проверь:
   `grep -c "{{" prompt.txt`
   Результат должен быть `0`.
4. Диспатч:
   `codex-companion.mjs task --background --write --effort medium`
5. Мониторь выполнение по шаблону из `skills/codex-invocation/SKILL.md`.
6. Получи итог через fetch/result.

## Фаза 4 -- Верификация

1. Прочитай отчёт Codex.
2. Проверь, что:
   - raw-копии созданы в `.vassal/raw/evidence-{дата}/`
   - зеркала созданы
   - `.vassal/index.yaml` валиден
   - в отчёте нет операций вне согласованного плана
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-add-evidence.md`:
   prompt + отчёт Codex.
4. Покажи Сюзерену резюме: сколько файлов обработано, сколько записей добавлено в индекс, есть ли `needs_manual_review`.

## Fallback без Codex

Если Codex недоступен:

- Предупреди Сюзерена: `Codex недоступен, выполняю apply-фазу напрямую`.
- Выполни apply-фазу самостоятельно по шагам из `prompts/file-executor-add-evidence.md`.
- OCR и извлечение текста запускай через:
  `python3 [PLUGIN_ROOT]/scripts/extract_text.py`
- Лог сохрани в `.vassal/codex-logs/{дата-время}-add-evidence-fallback.md`.

## Формат имени файла

```
ГГГГ-ММ-ДД Отправитель Описание.расширение
```

См. `shared/conventions.md` для полных правил.

## Идемпотентность

Перед обработкой проверь по `origin.name` + `origin.intake_batch` в `index.yaml`, не обработан ли файл ранее. Если обработан -- пропусти.

## Граничные случаи

- Те же, что в intake (пустой архив, нулевой размер, поврежденная кодировка, большой файл).
- Документ дублирует уже имеющийся (по содержимому) -> сообщи в preview, спроси Сюзерена.
