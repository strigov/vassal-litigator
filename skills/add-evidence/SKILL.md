# add-evidence -- Прием дополнительных доказательств

Прием и систематизация дополнительных документов от клиента по уже инициализированному делу. Работает по принципу **preview -> apply**. Отличие от intake: дело уже содержит материалы, индекс заполнен, контекст дела известен.

## Предварительные условия

- Дело инициализировано: `.vassal/case.yaml` существует. Если нет -- предложи `/vassal-litigator:init-case`.
- Intake уже выполнен (есть записи в `index.yaml`). Если нет -- предложи сначала `/vassal-litigator:intake`.
- Папка `Входящие документы/` существует.

## Алгоритм

### Фаза 1 -- Сохранение и обработка

1. Прочитай `.vassal/case.yaml` и `.vassal/index.yaml` -- тебе нужен контекст дела, стороны, уже известные документы.
2. **Источник файлов -- `Входящие документы/`**. Если папка пуста -- сообщи: «Папка "Входящие документы/" пуста. Положите туда файлы и запустите add-evidence снова.»
3. Скопируй все файлы в `.vassal/raw/evidence-{ГГГГ-ММ-ДД}/`.
4. Зафиксируй для каждого: исходное имя, дату получения, пакет (`evidence-{дата}`).
5. **OCR pipeline** -- как в intake:
   - PDF с текстовым слоем -> извлеки текст программно
   - DOCX -> Read tool / python-docx
   - Изображения / сканы -> tesseract, при низком качестве -- LLM fallback (Read tool)
6. **Категоризация** с учетом контекста дела:
   - Тип документа (из таксономии -- см. `shared/conventions.md`)
   - Дата, стороны
   - Назначение: для приобщения к делу? информационный?

### Фаза 2 -- Preview

7. Определи целевое расположение каждого файла:
   - Процессуальный документ для приобщения -> `{ГГГГ-ММ-ДД} {Сторона} - {действие}/`
   - Дополнительное доказательство к существующей категории -> `Материалы от клиента/{подпапка}/`
   - Назначение неясно -> `Материалы от клиента/Иное/` (с пометкой в preview)
8. Покажи Сюзерену план:

```
Пакет: evidence-2026-04-05 (3 файла)

Материалы от клиента/Платёжные документы/
  выписка.pdf -> 2026-01-15 Банк Выписка по счету за декабрь 2025.pdf

Материалы от клиента/Переписка/
  письмо.pdf -> 2026-03-20 ООО Ромашка Письмо-требование об оплате.pdf

ТРЕБУЮТ ПРОВЕРКИ:
  фото.jpg -> 0000-00-00 Неизвестный Документ.jpg (не удалось определить)

Связь с существующими документами:
  - Выписка может подтвердить/опровергнуть факт оплаты по Спецификации №2

После обработки файлы из «Входящие документы/» -> «На удаление/»
(оригиналы -> .vassal/raw/evidence-2026-04-05/)
```

9. Дождись подтверждения. Если Сюзерен корректирует -- внеси правки.

### Фаза 3 -- Apply

10. Создай подпапки (если нужны новые).
11. Скопируй файлы с новыми именами в целевые папки.
12. Создай md-зеркала в `.vassal/mirrors/` (делегируй Haiku-субагентам, model: "haiku").
13. Обнови `.vassal/index.yaml`:
    - Добавь записи с `source: client`
    - Заполни `origin` (исходное имя, дата, пакет `evidence-{дата}`)
    - Обнови `next_id`
14. Добавь запись в `.vassal/history.md`.
15. **Очистка «Входящие документы/»** -- перемести обработанные файлы в «На удаление/»:
    ```bash
    cp "Входящие документы/файл.pdf" "На удаление/файл.pdf"
    : > "Входящие документы/файл.pdf"
    ```

## Формат имени файла

```
ГГГГ-ММ-ДД Отправитель Описание.расширение
```

См. `shared/conventions.md` для полных правил.

## Идемпотентность

Перед обработкой проверь по `origin.name` + `origin.intake_batch` в `index.yaml`, не обработан ли файл ранее. Если обработан -- пропусти.

## Граничные случаи

- Те же, что в intake (пустой архив, нулевой размер, поврежденная кодировка, большой файл).
- Документ дублирует уже имеющийся (по содержимому) -> сообщи в preview, спроси Сюзерена.
