# Update-Index — Обновление и верификация индекса

Сканирует файловую систему дела и приводит `index.yaml` в соответствие с реальным состоянием файлов. Находит новые файлы без записи в индексе, «осиротевшие» записи без файлов, устаревшие зеркала.

## Предварительные условия

- Существует `.vassal/index.yaml`
- Существует `.vassal/case.yaml`

## Алгоритм

### Шаг 1 — Сканирование

1. Прочитай `.vassal/index.yaml`.
2. Просканируй файловую систему дела — все файлы в:
   - `Материалы от клиента/` (рекурсивно)
   - Все процессуальные папки `{ГГГГ-ММ-ДД} *` (рекурсивно)
   - Корень папки дела (кроме `.vassal/` и служебных файлов)
3. Игнорируй: `.vassal/` (внутренности), `На удаление/`, `Входящие документы/`, `.DS_Store`, `Thumbs.db`, `Таблица документов.xlsx`.

### Шаг 2 — Сравнение

4. Для каждого файла в файловой системе — проверь, есть ли запись в `index.yaml` (по полю `file`).
5. Для каждой записи в `index.yaml` — проверь, существует ли файл на диске.
6. Для каждого файла с записью — проверь, не устарело ли зеркало (файл изменился после `last_verified`).

Результат — три списка:
- **Новые файлы** (есть на диске, нет в индексе)
- **Осиротевшие записи** (есть в индексе, нет на диске)
- **Устаревшие зеркала** (файл изменился)

### Шаг 3 — Preview

7. Покажи Сюзерену результат сканирования:

```
Результат проверки индекса:

Файлов в индексе: 15
Файлов на диске: 17
Зеркал актуальных: 14

Новые файлы (нет в индексе):
  - Материалы от клиента/Иное/заметка.txt
  - 2026-04-01 Истец - пояснения/пояснения.pdf

Осиротевшие записи (файл не найден):
  - doc-005: Материалы от клиента/Переписка/старое_письмо.pdf

Устаревшие зеркала:
  - doc-003: mirrors/doc-003.md (файл изменён 2026-04-01)
```

8. Предложи действия:
   - Новые файлы → «Добавить в индекс и создать зеркала?»
   - Осиротевшие → «Удалить из индекса или файл был перемещён? Подскажи.»
   - Устаревшие → «Пересоздать зеркала?»

9. Дождись подтверждения.

### Шаг 4 — Apply

10. **Новые файлы**: для каждого —
    - Присвой ID (`doc-{next_id}`)
    - Извлеки текст (OCR pipeline)
    - Создай md-зеркало
    - Добавь запись в index.yaml
    - Определи тип, дату, стороны (делегируй Haiku при необходимости)

11. **Осиротевшие записи**: по указанию Сюзерена —
    - Удалить запись из индекса
    - Или обновить путь, если файл перемещён
    - Если запись удалена — переместить md-зеркало в `На удаление/` (не `rm`, см. `shared/conventions.md`)

12. **Устаревшие зеркала**: пересоздай md-зеркало (делегируй Haiku-субагенту).

13. Обнови `last_updated` и `last_verified` в index.yaml.
14. Добавь запись в `.vassal/history.md`.

### Шаг 5 — Верификация целостности

15. Финальная проверка:
    - Каждая запись в индексе имеет зеркало в `.vassal/mirrors/`
    - Каждое зеркало имеет валидный frontmatter (id, source, type)
    - Нет дубликатов ID
    - `next_id` корректен (больше максимального существующего ID)
    - Нет дубликатов путей в `file`

Сообщи о найденных проблемах.

## Идемпотентность

Повторный запуск безопасен — уже синхронизированные файлы не будут обработаны повторно.

## Граничные случаи

- Файл переименован вручную → старая запись станет «осиротевшей», новое имя — «новым файлом». Предложи Сюзерену связать их (обновить путь в существующей записи).
- Пустой индекс → предложи запустить intake
- Файл в `.vassal/raw/` появился без обработки → это нормально, raw/ не индексируется
