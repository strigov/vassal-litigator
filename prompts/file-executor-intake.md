## Вход

- `[CASE_ROOT]` = `{{case_root}}`
- `[PLUGIN_ROOT]` = `{{plugin_root}}`
- batch: `{{batch_name}}`
- согласованный план: `{{plan_body}}`
- дополнительные ограничения: `{{extra_constraints}}`

## Задача

Исполни подтверждённый intake-план буквально и по шагам. Не придумывай новых файловых операций и не меняй структуру дела вне переданного плана.

1. Получи план.
   `{{plan_body}}` — это уже согласованный план от Claude-main. Не улучшай его, не расширяй и не меняй последовательность действий. Если план противоречив, неполный или в нём не хватает обязательных данных для буквального исполнения, сразу остановись со статусом `NEEDS_CONTEXT`.
2. Копирование оригиналов в raw.
   Скопируй каждый исходный файл из его текущего расположения в `[CASE_ROOT]/.vassal/raw/intake-{ГГГГ-ММ-ДД}/`, сохраняя исходное имя файла. Используй `cp`, не `mv`. Папку создай, если её нет. В `.vassal/raw/` на этом шаге допустимо только копирование оригиналов; никаких других записей туда не делай.
3. OCR и извлечение текста.
   Для каждого файла из пакета вызови:
   `python3 [PLUGIN_ROOT]/scripts/extract_text.py --file "путь_к_файлу" --output "путь_к_tmp"`
   PDF с текстовым слоем `extract_text.py` обрабатывает напрямую. Изображения и сканы обрабатывай через tesseract внутри `extract_text.py`. Для каждого файла зафиксируй `extraction_method` и `confidence`.
4. Создание md-зеркал.
   Для каждого файла создай зеркало по шаблону `[PLUGIN_ROOT]/shared/mirror-template.md` в `.vassal/mirrors/doc-{NNN}.md`. ID бери только из переданного плана. Полностью заполни frontmatter: `id`, `title`, `date`, `doc_type`, `parties`, `source_file`, `origin_name`, `intake_batch`, `extraction_method`, `confidence`. Тело зеркала — извлечённый текст: первые 10 страниц или первые 20000 символов, что наступит раньше.
5. Обновление `index.yaml`.
   Для каждого файла добавь запись в `.vassal/index.yaml` с полями: `id`, `title`, `date`, `doc_type`, `parties`, `file` (целевой путь после переименования), `mirror`, `origin` (`name` + `date` + `batch`), `extraction_method`, `confidence`, `needs_manual_review` (`true`, если `confidence < 0.7` или распознавание не удалось), `mirror_stale: false`. Обнови `next_id`. После правок выполни:
   `python3 -c "import yaml; yaml.safe_load(open('.vassal/index.yaml'))"`
   YAML должен остаться валидным.
6. Перемещение файлов.
   Создай целевые папки только из переданного плана. Скопируй каждый файл в `[CASE_ROOT]/Материалы от клиента/Подпапка/Новое имя.расш`. Формат нового имени берётся из плана: `ГГГГ-ММ-ДД Отправитель Описание.расш`. Расширение исходного файла не меняй.
7. Очистка `Входящие документы/`.
   Для каждого успешно обработанного файла выполни:
   `cp "Входящие документы/файл.pdf" "На удаление/файл.pdf"`
   `: > "Входящие документы/файл.pdf"`
   Если в `На удаление/` уже есть файл с таким именем, добавь суффикс `(2)`, `(3)` и далее по порядку. Если файл не был успешно обработан, оставь его в `Входящие документы/` без изменений.
8. Запись `history.md`.
   Добавь в `.vassal/history.md` запись: дата, операция `intake`, список обработанных файлов, количество новых записей в индексе.
9. Валидация итога.
   Всегда выполни финальную проверку:
   `python3 -c "import yaml; d=yaml.safe_load(open('.vassal/index.yaml')); print('OK:', len(d['documents']), 'records')"`
   Результат проверки обязательно включи в отчёт, даже если один из предыдущих шагов завершился ошибкой.

Дисциплина исполнения:

- Не используй `rm`. Для удаления из пользовательских папок допустимы только `cp` + `: >`.
- Не пиши в `.vassal/raw/` ничего, кроме копирования оригиналов на шаге 2.
- Все пути к скриптам и shared-файлам плагина используй только через `[PLUGIN_ROOT]/...`, никогда не через относительные `./scripts/...`.
- Каждый шаг обязан иметь отдельную строку в `EXECUTION_LOG`.
- Если любой шаг с 1 по 8 завершился ошибкой, верни статус `BLOCKED`, не переходи к следующему шагу и явно зафиксируй точку остановки. Исключение: шаг 9 выполняется всегда.
